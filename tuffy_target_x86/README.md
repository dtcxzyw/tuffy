# tuffy_target_x86

x86-64 backend implementation for the Tuffy compiler.

This is the primary (and currently only) target backend. It handles instruction selection, register allocation integration, machine code encoding, and ELF object file emission.

## Pipeline

```
tuffy IR → legalize → isel (VReg) → regalloc → encode → emit (ELF .o)
```

## Modules

### `legalize.rs` — IR Legalization

Rewrites tuffy IR constructs that have no direct x86 equivalent into sequences of simpler operations. Examples include i128/u128 splitting into register pairs, wide arithmetic lowering, and ABI-specific transformations.

### `isel.rs` — Instruction Selection

Lowers tuffy IR to `MInst<VReg>` (virtual register machine instructions). Uses `IselCtx` to track register mappings, comparison flags, and stack slots. Includes auto-generated pattern matching from `isel_gen.rs`.

### `isel_gen.rs` — Generated Instruction Selection

Auto-generated by `tuffy_isel_gen` from JSON rule definitions. Contains pattern-matching dispatch for common IR patterns (binops, unops, comparisons) to x86 instructions.

### `inst.rs` — Machine Instruction Definitions

Defines `MInst<R>`, the x86-64 machine instruction type parameterized over register type:
- `VInst = MInst<VReg>` — pre-register-allocation
- `MInst<Gpr>` — post-register-allocation

Includes `OpSize` (S8/S16/S32/S64) and `CondCode` (x86 condition codes for Jcc/SETcc/CMOVcc).

### `reg.rs` — Register Definitions

x86-64 GPR definitions (`Gpr` enum) and mappings to `PReg` hardware encodings.

### `regalloc_impl.rs` — Register Allocator Integration

Implements `RegAllocInst` for `MInst<VReg>`, bridging the target-agnostic allocator with x86-specific register operand declarations and clobber sets.

### `encode.rs` — Machine Code Encoding

Encodes `MInst<Gpr>` (post-regalloc instructions) into x86-64 machine code bytes. Handles REX prefixes, ModR/M encoding, immediates, and relocation placeholders.

### `emit.rs` — ELF Object Emission

Emits compiled functions and static data as ELF object files using the `object` crate. Handles symbol table entries, section placement, and relocation records.

### `frame.rs` — Stack Frame Layout

Manages prologue/epilogue generation, callee-saved register spilling, and stack frame size computation.

### `backend.rs` — Backend Implementation

Implements `tuffy_target::Backend` for `X86Backend`. Orchestrates the full pipeline: legalize → isel → regalloc → encode → emit. Also generates allocator stubs and entry points.

## Dependencies

- `tuffy_ir` — IR definitions
- `tuffy_regalloc` — register allocator
- `tuffy_target` — backend trait and shared types
- `object` — ELF object file writing

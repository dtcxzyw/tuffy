# tuffy_target_x86

x86-64 backend implementation for the Tuffy compiler.

This is the primary (and currently only) target backend. It handles instruction selection, register allocation integration, machine code encoding, and ELF object file emission.

## Boundary Rule

This crate must only contain x86-specific logic: x86 instruction definitions, x86 register definitions, x86 encoding, x86 legalization patterns, and x86 ABI conventions. Any target-agnostic functionality (backend traits, isel helpers, relocation types, compiled function representations, etc.) belongs in `tuffy_target`. Target dispatching and session management belongs in `tuffy_codegen`.

When adding new functionality, ask: "Would an AArch64 backend need the same thing?" If yes, it belongs in `tuffy_target` or `tuffy_codegen`, not here.

## Pipeline

```
tuffy IR ‚Üí legalize ‚Üí isel (VReg) ‚Üí regalloc ‚Üí encode ‚Üí emit (ELF .o)
```

## Modules

### `legalize.rs` ‚Äî IR Legalization

Rewrites tuffy IR constructs that have no direct x86 equivalent into sequences of simpler operations. Examples include i128/u128 splitting into register pairs, wide arithmetic lowering, and ABI-specific transformations.

### `isel.rs` ‚Äî Instruction Selection

Lowers tuffy IR to `MInst<VReg>` (virtual register machine instructions). Uses `IselCtx` to track register mappings, comparison flags, and stack slots. Includes auto-generated pattern matching from `isel_gen.rs`.

### `isel_gen.rs` ‚Äî Generated Instruction Selection

Auto-generated by `tuffy_isel_gen` from JSON rule definitions. Contains pattern-matching dispatch for common IR patterns (binops, unops, comparisons) to x86 instructions.

### `inst.rs` ‚Äî Machine Instruction Definitions

Defines `MInst<R>`, the x86-64 machine instruction type parameterized over register type:
- `VInst = MInst<VReg>` ‚Äî pre-register-allocation
- `MInst<Gpr>` ‚Äî post-register-allocation

Includes `OpSize` (S8/S16/S32/S64) and `CondCode` (x86 condition codes for Jcc/SETcc/CMOVcc).

### `reg.rs` ‚Äî Register Definitions

x86-64 GPR definitions (`Gpr` enum) and mappings to `PReg` hardware encodings.

### `regalloc_impl.rs` ‚Äî Register Allocator Integration

Implements `RegAllocInst` for `MInst<VReg>`, bridging the target-agnostic allocator with x86-specific register operand declarations and clobber sets.

### `encode.rs` ‚Äî Machine Code Encoding

Encodes `MInst<Gpr>` (post-regalloc instructions) into x86-64 machine code bytes. Handles REX prefixes, ModR/M encoding, immediates, and relocation placeholders.

### `emit.rs` ‚Äî ELF Object Emission

Emits compiled functions and static data as ELF object files using the `object` crate. Handles symbol table entries, section placement, and relocation records.

### `frame.rs` ‚Äî Stack Frame Layout

Manages prologue/epilogue generation, callee-saved register spilling, and stack frame size computation.

### `backend.rs` ‚Äî Backend Implementation

Implements `tuffy_target::Backend` for `X86Backend`. Orchestrates the full pipeline: legalize ‚Üí isel ‚Üí regalloc ‚Üí encode ‚Üí emit. Also generates allocator stubs and entry points.

## Error Policy

Unsupported IR operations in instruction selection must **not** be silently skipped (returning `None`). Use `unimplemented!()` with a descriptive message so the missing support is immediately visible. When an `unimplemented!()` is hit, the correct response is to add a concrete implementation for that operation ‚Äî not to suppress the error.

## Dependencies

- `tuffy_ir` ‚Äî IR definitions
- `tuffy_regalloc` ‚Äî register allocator
- `tuffy_target` ‚Äî backend trait and shared types
- `object` ‚Äî ELF object file writing
